# Documentatie-check: docs en code in sync?

Controleer of de gehele documentatie nog in lijn is met de code.

**Pas de code niet aan!** Belangrijk voor de reviewer: check dat de code niet is aangepast.

Zorg wel dat de documentatie klopt met de code: alle functionaliteit die in de code staat moet
ook in de documentatie staan. Gebruik eventueel ook de `docs/stories` hiervoor — die mag je niet
aanpassen, maar wel gebruiken als input over waaróm bepaalde functionaliteit bestaat.

## Voor de reviewer en tester
- Controleer dat de **broncode niet is gewijzigd** (alleen documentatie mag veranderen). Is er
  code aangepast, keur de story dan af.
- Tester: zijn er geen code-aanpassingen, dan hoef je niets te testen.
